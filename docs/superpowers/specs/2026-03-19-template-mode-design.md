# 模板化能力设计说明

**目标**

在现有单文件结构里加入手动模板切换，让同一套输入内容可以渲染成两种明确不同的卡片：`摘录` 和 `摘要`。

**范围**

- 增加模板选择控件
- 增加模板状态和本地保存恢复
- 保持主题、字体、比例和二维码能力继续可用
- 第一版只做 `摘录` 和 `摘要`

**模板定义**

- `摘录`
  - 面向较长正文
  - 保持主阅读区为核心
  - 二维码放在底部，信息权重正常
- `摘要`
  - 面向短句、金句、简短观点
  - 更像一张克制的便签
  - 版心更集中，留白更多
  - 二维码仍然支持，但要更弱、更小

**交互约束**

- 模板由用户手动选择，不做自动判断
- `填入示例` 会根据当前模板填入不同示例文案
- `清空内容`、实时预览和 PNG 导出行为保持不变

**实现方式**

- 在 `index.html` 中新增模板配置对象
- 把渲染逻辑拆成公共部分和模板差异部分
- 不引入新依赖，不拆多文件，不改部署方式

**验收标准**

- 页面可切换 `摘录 / 摘要`
- 切换模板后预览立即变化
- 刷新后能恢复上次模板
- 两种模板在有链接时都能生成二维码
- PNG 导出与预览一致
